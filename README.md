# АРС БУДІНВЕСТ — лендінг

Статичний лендінг на **[Astro](https://astro.build/)**. Без бекенду, без бази даних —
рендериться в чистий HTML/CSS, тож вантажиться миттєво й безкоштовно лягає на
**Cloudflare Pages**. Купити треба тільки домен.

## Чому Astro?

Для лендінга це найкращий вибір: 0 КБ JavaScript за замовчуванням, відмінне SEO,
вбудована оптимізація зображень, простий контент у JSON. React/Next.js тут — оверкіл
(зайвий рантайм, гірший за швидкістю та SEO для звичайної маркетингової сторінки).

## Команди

```bash
npm install      # встановити залежності (один раз)
npm run dev      # локальний сервер → http://localhost:4321
npm run build    # зібрати продакшн у dist/
npm run preview  # переглянути зібраний продакшн локально
```

## Структура

```
public/
  assets/              ← зображення (hero — поки плейсхолдер, решта — реальні фото)
    hero/  projects/  blocks/  equipment/
  favicon.svg
  robots.txt
  _headers             ← кеш/безпека для Cloudflare Pages
src/
  data/                ← УВЕСЬ текст і "товари" в JSON (міняєте тут)
    site.json          ← телефон, email, соцмережі, герой, CTA, смуга довіри
    services.json      ← блок "Ми пропонуємо"
    projects.json      ← картки проєктів будинків
    blocks.json        ← виробництво блоків
    equipment.json     ← оренда техніки
    features.json      ← "Чому обирають нас"
    process.json       ← "Як ми працюємо"
  components/           ← секції сторінки (.astro)
  layouts/Base.astro   ← <head>, SEO/OG-теги, шрифт
  pages/index.astro    ← збирає всі секції разом
  styles/global.css    ← кольори бренду та типографіка (дизайн-токени)
```

## Як міняти контент

- **Текст, ціни, телефон, "товари"** — редагуєте відповідний файл у `src/data/*.json`.
  Наприклад, щоб додати проєкт будинку — додайте об'єкт у `projects.json`.
- **Кольори** — `src/styles/global.css`, секція `:root` (помаранчевий, зелений тощо).
- **Зображення** — кладіть у `public/assets/...` і вкажіть шлях у полі `image` відповідного
  JSON-файлу (бажано `.webp`). Усі фото вже реальні, окрім головного банера (`hero/`).

## Деплой на Cloudflare Pages (безкоштовно)

1. Залийте проєкт у GitHub-репозиторій.
2. Cloudflare Dashboard → **Workers & Pages** → **Create** → **Pages** → **Connect to Git**.
3. Налаштування збірки:
   - **Framework preset:** `Astro`
   - **Build command:** `npm run build`
   - **Build output directory:** `dist`
4. Після першого деплою прив'яжіть свій домен: **Custom domains** → додати домен.
5. У `astro.config.mjs` замініть `site: 'https://arsbudinvest.ua'` на ваш реальний домен,
   і в `public/robots.txt` оновіть адресу `Sitemap:`.

> Альтернатива без GitHub: `npm run build`, потім перетягніть теку `dist/` у Cloudflare
> Pages (Direct Upload).

## Відомий компроміс доступності

Білий текст на фірмовому помаранчевому (`#e5611e`) дає контраст ≈3.47:1 — нижче порогу
WCAG AA (4.5:1) для дрібного тексту. Кольори зафіксовані за вашим побажанням, тож я
**не змінював** палітру. Якщо потрібна сувора відповідність AA, є два варіанти (скажіть —
зроблю): (1) трохи затемнити помаранчевий до ~`#c2521a`, або (2) збільшити кегль тексту
кнопок до ≥19px (тоді він кваліфікується як «великий» і поріг падає до 3:1).

## Кнопки заклику до дії

Кнопки "Отримати кошторис" і "Замовити дзвінок" — це `tel:`-посилання (`href`
у `site.json` → `phoneHref`), тож натискання одразу ініціює дзвінок (на мобільному —
відкриває набір номера, як клік по телефону).

Якщо в майбутньому захочеться **форму заявки** (залишити ім'я/номер для зворотного
дзвінка) замість прямого дзвінка — найпростіші варіанти
підключення заявок на статиці: [Web3Forms](https://web3forms.com/),
[Formspree](https://formspree.io/) або Cloudflare Pages Functions. Скажіть, який варіант
бажаєте — підключу.
